# 火星文翻译器 · UI 设计方案 V2

> **角色**：UI Designer（界面设计专家）
> **目标**：在「桌面优先 Web + 活泼品牌化」约束下，对现有火星文翻译器做**体验升级**——保留其完整功能（双向翻译、历史、收藏、分享卡、7 类规则配置、确定性引擎），重做信息架构、视觉语言与组件体系。
> **配套视觉稿**：`mars-translator-ui-design.html`（可交互原型，含明暗主题 / 方向切换 / 复制分享收藏 / 响应式）

---

## 1. 设计目标与原则

| 原则 | 落地方式 |
|---|---|
| **美观 × 效率并重** | 宇宙渐变品牌色 + 圆润卡片营造活力；双栏工作台让「输入↔结果」同屏对照，减少滚动与来回切换 |
| **贴合心智模型** | 翻译器用户期待「左右对照 / 方向切换 / 即时结果」，沿用此范式而非发明新交互 |
| **一致性** | 全部视觉来自统一设计令牌（色 / 字 / 距 / 圆角 / 阴影），组件可复用，杜绝碎片化 |
| **可访问性（WCAG AA）** | 正文对比度 ≥ 4.5:1，焦点可见，语义化标签，44px 触控目标，尊重 `prefers-reduced-motion` |
| **平台规范** | 桌面端左侧固定导航 + 顶栏；移动端底部标签栏——符合主流 Web/App 导航惯例 |

---

## 2. 信息架构（IA）与导航结构

### 2.1 信息架构树

```
火星文翻译器
├── 翻译工作台（默认首页）
│   ├── 方向切换：汉 → 火 / 火 → 汉（含一键交换）
│   ├── 源文本输入（示例短语 / 清空 / 字数统计）
│   ├── 火星文结果（复制 / 分享卡片 / 收藏）
│   └── 转换规则配置（风格模板 · 强度 · 7 类规则开关 · 字符集白名单）
├── 历史记录
│   ├── 搜索 + 排序（时间 / 方向）
│   └── 单条：复用翻译 / 收藏 / 删除
├── 我的收藏
│   └── 收藏夹列表（原 localStorage，补一个独立视图，与历史解耦）
└── 设置
    ├── 外观（主题 / 强调色）
    ├── 翻译默认值（默认方向 / 自动存历史 / 长文分批）
    └── 关于（版本 / 引擎说明）
```

> **关键改动**：原版「收藏」仅藏在结果卡按钮里、无独立页。本方案把「收藏」提升为**一级导航**，与历史并列，符合「常用功能应触手可及」的心智模型；同时规则配置从独立页**下沉为翻译页内联面板**，因为用户调规则是为了看效果，同屏即时预览比跳页更顺。

### 2.2 导航结构（桌面 / 移动）

- **桌面（≥1024px）**：左侧固定 **侧边栏**（256px），含品牌 Logo、主导航（翻译/历史/收藏/设置）、底部主题切换。主区顶部为 **顶栏**（页面标题 + 面包屑 + 全局操作）。
- **移动（<1024px）**：侧边栏变为**底部标签栏**（图标 + 文字），顶栏保留标题。符合移动端拇指热区与平台习惯。
- **导航状态**：当前页高亮（品牌渐变软底 + 强调色文字），支持键盘 `Tab` 聚焦与 `Enter` 激活。

## 3. 关键页面交互流程

### 3.1 核心翻译流（主路径）
```
选择方向(汉→火/火→汉) ──┐
                         │
输入源文本 ──► 实时预览(防抖) ──► 结果卡渲染
                         │              │
                         │              ├─ 复制 ──► Toast「已复制」
                         │              ├─ 分享卡片 ──► Modal(Canvas PNG) ──► 下载
                         │              └─ 收藏 ──► 星标高亮 + 写入收藏
                         ▼
              自动沉淀历史(去重, 800ms 防抖)
```
- **方向交换**：点交换按钮或切 Tab 时，**框内文本自动对调**并重新翻译，保持连续心流（避免清空打断）。
- **空输入**：禁用操作 + 提示「请输入要转换的内容」；>1000 字按词边界分批并显示进度条（沿用引擎既有策略）。

### 3.2 规则配置流
```
展开规则面板 ──► 选风格模板(anime/retro/minimal/custom)
              ──► 调强度(低/中/高)
              ──► 开关 7 类规则
              ──► 填字符集白名单
                         │
                         ▼
              Zustand 配置变更 ──► 引擎重算 ──► 结果卡即时刷新（无需重载）
```

### 3.3 历史 / 收藏复用流
```
历史/收藏列表 ──► 点「复用」──► 跳翻译页 + 回填文本 + 同步方向 ──► 自动聚焦输入
```

## 4. 色彩体系规范

### 4.1 品牌与语义色（明 / 暗双主题令牌）

| 用途 | Token | 浅色 | 暗色 | 说明 |
|---|---|---|---|---|
| 主品牌 | `--brand` | `#FF3E7F` | `#FF5C95` | 品红 |
| 次品牌 | `--brand-2` | `#7C5CFF` | `#9C86FF` | 紫 |
| 品牌渐变 | `--brand-grad` | `135° #FF3E7F→#7C5CFF` | 同 | Logo / 主按钮 / 激活态 |
| 强调（收藏） | `--accent` | `#FF9F43` | `#FFB45E` | 星标暖色 |
| 成功 | `--success` | `#15B97A` | `#2FD493` | |
| 警告 | `--warning` | `#F5A623` | `#FFC04D` | |
| 危险 | `--danger` | `#FF4D4F` | `#FF6B6E` | |
| 信息 | `--info` | `#3D8BFF` | `#5BA0FF` | |

> **令牌权威性说明**：上表「浅色 / 暗色」双主题为**唯一权威取值**。原型与最终实现必须严格采用对应主题的值——例如暗色下 `--brand` 取 `#FF5C95`、`--brand-2` 取 `#9C86FF`，**不得混用浅色值**。视觉稿 `mars-translator-ui-design.html` 的 `[data-theme="dark"]` 已据此补全暗色令牌。

### 4.2 中性色阶（浅色）
`--text #1B1730` / `--text-sub #6A6385`(对比度≈5.0:1 ✅) / `--text-muted #9A93B2` / `--surface #FFFFFF` / `--surface-2 #F6F3FF` / `--border #ECE7F5` / `--bg #FBF8FF`。

### 4.3 对比度与可访问性
- 正文 `#1B1730` on `#FFFFFF` ≈ **15:1**；次级 `#6A6385` on `#FFFFFF` ≈ **5.0:1**（满足 AA 4.5:1）。
- 主按钮白字 on 品牌渐变：品红段 ≈ 4.6:1、紫段 ≈ 5.3:1，均达标。
- 暗色下文字提亮至 `#F3EEFF`，次级 `#ABA0CE`，保证可读。

## 5. 字体排版规范

| 项 | 规范 |
|---|---|
| 主字体 | `'Inter', 'PingFang SC', 'Microsoft YaHei', system-ui`（中文优先 PingFang/雅黑，西文 Inter） |
| 等宽（代码/坐标） | `'JetBrains Mono', ui-monospace` |
| 字号阶梯 | 12 / 13 / 14 / 16 / 18 / 20 / 24 / 30 / 38 px（4 基准 + 视觉节奏） |
| 字重 | 400 / 500 / 600 / 700 / 800 |
| 行高 | 正文 1.55–1.7；标题 1.2–1.3 |
| 使用建议 | 标题 700–800 + 紧字距；正文 400–500；按钮/标签 600 |

## 6. 间距 / 圆角 / 阴影 / 动效令牌

- **间距（4px 基准）**：4 / 8 / 12 / 16 / 20 / 24 / 32 / 40 / 48 / 64，全局统一节奏。
- **圆角**：`--r-sm 8` / `--r-md 12` / `--r-lg 16` / `--r-xl 20` / `--r-pill 999`（活泼感来自大圆角 + 胶囊）。
- **阴影**：`--sh-sm`（卡片静息）/ `--sh-md`（hover 抬升）/ `--sh-lg`（浮层/弹窗）/ `--sh-glow`（主按钮品牌辉光）。
- **动效**：`--t-fast 140ms`（悬停/开关）/ `--t-norm 260ms cubic-bezier`（页面/弹窗）；尊重 `prefers-reduced-motion` 时降级为无位移。

## 7. 组件一致性与可复用设计

所有组件基于同一令牌驱动，确保 95%+ 一致性。核心组件清单：

| 组件 | 变体 / 状态 | 复用点 |
|---|---|---|
| **Button** | primary（渐变）/ default / ghost / icon；尺寸 sm/base；状态 hover/active/disabled/focus-visible | 全站 CTA、工具栏、弹窗 |
| **Card** | 基础卡 / 配置卡 / 结果卡 / 记录卡 | 工作台、历史、设置分区 |
| **Segmented** | 方向切换 / 强度 / 主题 | 互斥单选场景统一控件 |
| **Chip** | 风格模板 / 示例短语；active = 渐变 | 多选/快捷入口 |
| **Switch** | 7 类规则开关 / 设置开关 | 布尔配置统一控件 |
| **Tag** | 方向标签 / 品牌标签 | 元信息标记 |
| **Textarea/Input** | 源输入 / 白名单 / 搜索；focus 品牌环 | 所有文本录入 |
| **Toast** | 复制/收藏/下载反馈 | 全局轻提示 |
| **Modal** | 分享卡弹窗 | 浮层统一容器 |
| **NavItem** | 侧栏/底栏；active 高亮 | 导航统一 |

**复用纪律**：
1. 不允许在组件内硬编码颜色/字号，一律引用令牌。
2. 新增控件先查组件库，优先扩展而非新建。
3. 状态样式（hover/active/focus/disabled）必须成对定义，避免「能点但无反馈」。

## 8. 响应式适配策略

| 断点 | 布局 |
|---|---|
| **桌面 ≥1024px** | 侧边栏 + 顶栏；工作台**双栏**（输入｜结果）+ 规则面板；列表多列网格。完整功能集。 |
| **平板 768–1023px** | 侧栏转底部标签栏；双栏 → **单栏堆叠**；规则面板转全宽；列表 2 列。 |
| **手机 <768px** | 底部标签栏；单栏；分段控件缩排；规则开关单列；列表单列；触控目标 ≥44px。 |

- 采用 **Mobile First + 渐进增强**：基础单列，≥1024px 升级为双栏工作台。
- 容器最大宽 1180px 居中，留白呼吸感。
- 背景装饰光斑 `position: fixed` 不随内容滚动，性能友好。
