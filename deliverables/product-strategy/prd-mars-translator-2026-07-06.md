# 第一部分：功能规格书（PRD，析客）

## 1. 产品目标（3 个清晰、正交的目标）

- **G1 能力目标（差异化价值）**：提供汉语↔火星文、英语↔火星文的双向高保真转换，并支持规则可配置，解决行业「只转不还原、规则写死」的第一痛点。衡量护栏：火→汉反向还原准确率作为核心质量底线。
- **G2 增长目标（北极星闭环）**：以「翻译→复制/分享/收藏」操作闭环驱动复访与病毒增长，支撑北极星指标周成功翻译操作数（WSTA）。衡量：整体访问→操作转化率、分享率、病毒系数 K。
- **G3 平台目标（产品化扩展）**：沉淀可配置规则体系与风格模板，为后续 API 嵌入与 UGC 社区预留架构，避免一次性工具化。衡量：规则可复现率、配置项覆盖度。

三者正交关系：G1 解决「能不能用 / 好不好用」，G2 解决「用不用得久 / 传不传得开」，G3 解决「能不能长大 / 接得进生态」。

## 2. 用户故事（3-5 个场景）

- **US1 游戏起名（early adopter·Z世代社交玩家）**：小莫（16）想给新游戏角色起一个「圈内人」风格的火星文昵称。她输入「可爱の猫」，选择汉→火，实时看到多个风格结果，一键复制最酷的一个到游戏。
- **US2 批量签名收藏（early adopter·网名签名创作者）**：阿岚（20）需要 10 条风格统一的个性签名。她开启多风格模板，批量生成并逐条「收藏」，日后从历史复用。
- **US3 还原旧消息（怀旧90后）**：老周（33）收到朋友发来的一段火星文祝福看不懂。他切到火→汉入口粘贴，秒级还原成通顺汉语，会心一笑并收藏留念。
- **US4 弹幕/帖子批量创作（内容创作者）**：小鹿（27）运营粉丝群，需批量生成火星文弹幕。她生成后导出「结果图片分享卡」发到群，引发转发。
- **US5 二次元风格表达（亚文化）**：阿茶（18）偏好颜文字+片假名混排风格，在设置中开启「日语假名借用 + emoji 穿插」，获得符合审美的输出。

## 3. 用户研究洞察（来自瑞思）

**5 类核心画像**：
1. Z世代社交玩家（15–22）：社交娱乐 / 圈层归属，趣味 > 准确但极敏感「像不像圈内人」。
2. 亚文化/二次元（16–25）：颜文字 / 片假名混用，重视视觉美感与符号丰富度。
3. 网名签名创作者（18–30）：独特性要求极高，需批量生成 + 收藏。
4. 怀旧90后/00初（28–38）：还原当年写法，需历史模板库。
5. 内容创作者/社群运营（22–35）：批量 + 可复制 + 一键分享。

**场景地图情绪曲线**：高期待 → 输入略降 → 结果两极（酷=尖峰 / 丑=骤降）→ 分享峰值。关键流失点：①输入繁琐 ②结果不够味 ③复制分享不顺。

**7+1 核心洞察**：
1. 「像不像圈内人」>「准不准」——对准确率容忍高，对地道/够酷极敏感。
2. 双向方向相反（汉→火表达 / 火→汉理解），用户重叠低，入口须区分。
3. 英语↔火是小众长尾差异化，竞品少见。
4. 收藏/历史是复访核心钩子（ID/签名具反复使用属性）。
5. 可分享性直接决定增长（火星文是社交货币）。
6. 自定义符号/风格是强长尾需求（高级用户想改规则、混排颜文字）。
7. 怀旧人群是低成本 UGC 源（愿贡献经典模板）。
8.【校准矛盾】核心群「要个性 vs 要可读 / 要能还原」——既想酷炫独特，又需对方读懂且能双向还原。串联洞察 1 与 2：个性过头会不可读，故「风格可控」比「越花越好」更重要；**火→汉还原是 P0 核心需求**。

**Early adopters 结论**：画像①+③合并 = 社交娱乐表达型用户（昵称/签名/弹幕/游戏起名），MVP 优先服务；长文翻译降级为衍生场景。

**体验短板**：单向无法还原（行业最大痛点）、规则写死不可控、准确率低不保意、无历史与收藏、广告泛滥界面旧、移动与 API 缺位、跨语言空白、社交场景割裂。

**SWOT**：
- S：双向还原 + 可配置 + 英火空白 + 产品化体验闭环。
- W：冷启动无品牌、小众天花板、移动/API 非首期、规则引擎研发成本。
- O：怀旧经济与「非主流复兴」、昵称/签名/弹幕/游戏起名社交场景、可嵌入输入法与社交平台的 B 端 API、二次元与海外华语圈表达。
- T：大厂输入法若内置同类功能将碾压流量、小众需求商业化有限、开源库可免费复刻核心逻辑、微信/QQ 对特殊字符的政策限制。

**差异化机会**：双向高保真还原、多语言英↔火、规则可配置、社交场景嵌入、产品化体验闭环。

## 5. 数据依据（来自数析，v1.2）

**北极星**：周成功翻译操作数 WSTA = 单周内完成翻译且发生「复制/分享/收藏」任一操作的次数。

**核心漏斗（[建议目标]）**：输入 ≥65% → 翻译 ≥90% → 操作 ≥45%；整体访问→操作 ≈26%。两处流失埋点：输入放弃率、结果页未操作离开率。双向入口双切片（语言向：汉↔火 vs 英↔火；方向：汉→火 vs 火→汉）。

**市场（双口径，[行业估算]）**：注意力峰值（梗爆发期百万级曝光，属注意力非留存）；可变现 DAU 稳态保守 1万–10万；TAM 偏小；脉冲式趋势靠话题借势。

**质量三指标**：
1. 词级编辑距离近似准确率（主指标） + BLEU 类 n-gram 重叠参考 + 字符级底线；
2. 反向还原准确率（火→汉）对称监控；
3. 规则可复现率 ≥99%（确定性映射）。

> **阈值约定（补全「规划底线」）**：① 「词级编辑距离近似准确率」的绝对基线值（即上文「规划底线」）由数析在 V1.0 回归基线建立时给定，发布采用**相对阻断**——较基线下降 >2pp 即阻断；② 「翻译成功率 ≥90%」是上线 / 灰度退出的**操作性门槛**（与质量三指标并列，见第二部分发布门槛），不属于质量三指标本身；③ 规则可复现率 ≥99% 为绝对硬门槛。三者共同构成质量护栏，避免「规划底线」悬空、与发布闸门混淆。

**回归基线规模**：CI 全量回归测试集基线 **≥2000 条**；月级人工抽检 **≥200 条**（与 Dev-Ready §7、Sprint 计划 T05 一致，避免基线规模口径不一）。

**回归机制**：规则更新后全量回归，词级准确率或反向还原准确率下降 >2pp 即阻断发布；周级自动回归、月级人工抽检 ≥200 条。

**增长与留存目标（[建议目标]）**：病毒系数 K ≥0.3、分享率 ≥25%、D7 ≥30% / D30 ≥12%、收藏率 ≥15% / 历史复用率 ≥20%；满意度双轨代理（分享+收藏为正向，二次编辑率为中性参与）。

**数据看板 Top10**：①WSTA ②DAU/WAU ③访问→操作转化 ④翻译成功率 ⑤分享率&K ⑥D7/D30 留存 ⑦收藏率/历史复用率 ⑧双向入口双切片 ⑨质量三指标 ⑩渠道来源 & 单会话翻译次数。

**风险清单**：①英语↔火星文需求是否真实（用语言向占比+留存验证）；②规则可配置是否提升留存（需 A/B）；③脉冲式流量能否沉淀；④质量-增长权衡（设准确率底线）；⑤平台政策风险（微信/QQ 特殊字符过滤压低裂变，列为市场规模天花板变量）。

## 7. 火星文规则体系（必含章节）

1. **同音字替换**：「我」→「涐/莪」；「的」→「嘚/の」；「你」→「祢」。
2. **形近字替换**：「人」→「亾」；「国」→「囻」；「爱」→「愛」。
3. **注音符号混入**：「好」→「好ㄅ」；「吃」→「吃ㄔ」；「吗」→「吗ㄇ」。
4. **拆字重组**：「强」→「弓虽」；「好」→「女子」；「明」→「日月」。
5. **emoji/符号穿插**：「可爱」→「可♡爱」；「开心」→「开✧心」；「加油」→「加★油」。
6. **日语假名借用**：「的」→「の」；「是」→「です」；「我」→「私」。
7. **拼音首字母缩写**：「什么」→「sm」；「谢谢」→「xx」；「喜欢」→「xh」。

**可配置维度**：①强度（替换密度低→高）；②字符集（启用规则族 / 繁体开关 / 符号白名单）；③风格模板（「二次元」「怀旧」「极简」等预设组合）。全部确定性映射，可复现率 ≥99%。

## 8. 边界情况处理（必含章节）

- **空输入**：结果区灰态提示「请输入要转换的内容」，禁用复制/分享/收藏。
- **超长文本**：>2000 字自动分块，显示进度条与分块序号，逐块返回，避免阻塞。
- **混合语言识别**：按字符 Unicode 区间判定中/英/符号，分别路由规则链，互不污染。
- **无法翻译字符**：原样保留并加轻量标记（虚线下划线），不参与映射，保信息不丢。

## 9. 技术可行性评估（必含章节）

**纯规则引擎 vs NLP 模型**：

| 维度 | 纯规则引擎+词典 | 引入 NLP 模型 |
|---|---|---|
| 确定性/可复现 | 高（≥99%） | 低（随机性，难复现） |
| 反向可控性 | 强（映射可逆） | 弱（需额外消歧） |
| 延迟 | 毫秒级 | 较高 |
| 配置灵活度 | 高（开关即时） | 低 |
| 冷启动成本 | 低 | 高（训练/标注） |
| 语义理解 | 弱 | 强 |

MVP 选**纯规则 + 核心词库**，准确率底线由质量三指标监控；NLP 仅作 V2.0 可选消歧。

**双向难点**：火星文不规范、多义性强、反向需上下文消歧。应对 = 确定性映射 + 词典优先 + 可选 NLP 消歧；无法消歧时回退「原样保留 + 标记」。

## 10. 数据与运营策略（必含章节）

- **词典建设**：手工标注核心词库（昵称/签名/圈层用语）+ 用户反馈众包（纠错即贡献）。
- **质量评估**：准确率测试集 + 全量回归（降 >2pp 阻断）+ 周自动 / 月人工抽检 ≥200。
- **增长策略**：社交场景嵌入（签名/网名生成器、弹幕模板）；病毒式传播（图片卡一键转发 + 来源水印）；内容化冷启动（怀旧模板 + 网感话题借势）。

## 1. 当前指标表现基线（来自数析，规划基准）

以下为 MVP 上线前的规划基准，产品尚未发布，全部数值以「[建议目标]」形式给出，作为路线图各阶段验收与迭代调优的对标线。

- **北极星 WSTA**：[建议目标] 首季稳态 ≥ 8 万次/周。
- **转化漏斗**：输入率 ≥65% → 翻译成功率 ≥90% → 操作完成率 ≥45%。
- **增长与留存**：K ≥0.3、分享率 ≥25%、D7 ≥30% / D30 ≥12%、收藏率 ≥15% / 历史复用率 ≥20%。

**规划含义**：漏斗显示「操作完成率」是最大漏点（45% vs 输入 65%、翻译 90%），故 MVP 资源优先压在「收藏/复访钩子」（R05/R06）与「结果图片分享卡」（R07），而非多语言扩展；英语方向（风险①）后置验证。

## 2. 用户反馈摘要（来自瑞思）

- **正向**：双向还原准确、可配置风格让同一条文案呈现多种「人设」，社交娱乐场景命中度高。
- **痛点**：长句混合语言时字符保留混乱、微信/QQ 对部分特殊符号过滤导致分享卡变形、词库覆盖不足时还原不唯一。
- **沉默流失点**：无收藏/历史复访时，用户翻译一次即走，缺乏回巢钩子——印证操作完成率为关键漏点。
- **需求演化四方向**：①翻译工具 → 表达工具；②视觉化（图片卡/字体/深色模式）；③社交化（分享/收藏/复用闭环）；④平台化（UGC 模板/批量起名，V2.0 承接）。

## 3. 竞品动态（来自竞析）

**六类现有形态**：纯转换站（单向）、输入法/插件式火星文键盘、表情包/颜文字聚合站、起名/昵称生成器、翻译类 App 的「趣味」附加模块、社区 UGC 模板平台。

**差异化空白**：双向还原（竞品多为单向）、英↔火（全行业空白，但需求真实性存疑）、可配置（竞品固定风格）、社交嵌入（结果图片卡 + 一键分享 + 收藏复访，竞品缺乏传播闭环）。

**结论**：MVP 以「双向还原 + 可配置风格 + 社交娱乐场景」占据双向还原与社交嵌入空白，英语与平台化作为后续差异化纵深。

## 5. 上线与迭代策略（来自路径）

**灰度发布路径（Web 优先）**：
1. 小流量内测（0→5%）：内部 + 早期 adopters 邀请码，验证核心翻译与还原准确率、分享卡渲染。
2. 分群灰度（5%→30%→100%）：按渠道分群（社交娱乐内容社区优先），观察漏斗各环节与 WSTA 斜率。
3. 全量：达标后开放公网，启动词库众包扩词。

**反馈收集与规则优化闭环**：灰度期每日回流失败样本（无法还原/乱码/过滤变形）→ 规则引擎热更新 → 词库周更 → 复用率与收藏率周环比监控。

**回滚条件（质量三指标回归阻断）**：任一版本上线后若触发以下任一，立即回滚并冻结新功能：
- 翻译成功率 < 90%（回归阈值）
- 确定性映射可复现率 < 99%
- 分享卡渲染失败率 > 3%（第三方字符过滤异常）

**商业化探索（后置）**：在 D30 留存与分享率稳定达标后，探索火星文周边（表情包生成、人设贴纸）、起名增值模板，避免早期干扰北极星增长。

## 6. 利益相关者沟通要点（按受众）

**高管视图**：差异化壁垒=双向还原确定性 + 可配置风格 + 社交传播闭环；北极星 WSTA 唯一对齐；TAM 偏小且受热点脉冲影响，以收藏/复访钩子平滑留存；英语验证时点=V1.0 上线 4 周内英火试入口数据，未达标即延期。

**工程视图**：架构=MVP 纯规则引擎 + 核心词库，NLP 仅 V2.0 可选消歧，主路径零联网大模型依赖；确定性映射 ≥99% 可复现；边界实现=分块处理、混合语言识别（中/英/符号分段）、无法翻译字符原样保留；质量三指标回归即回滚，纳入 CI 门禁。

**设计视图**：信息架构=双入口对称布局降低首屏认知成本；分享卡统一视觉模板 + 品牌水印 + 深色模式兼容；个性化=深色模式 + 字体大小；针对微信/QQ 特殊字符过滤采用「安全字符优先 + 兜底预览」展示策略。
