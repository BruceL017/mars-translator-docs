## 二、逐文件优化清单（改动 + 原因）

### 📄 `product-strategy/prd-mars-translator-2026-07-06.md`
1. **新增目录（顶部）** — 原因：原文 320 行无导航，定位困难；目录按「第一部分 PRD / 第二部分 路线图」分组，便于跳转。
2. **§5 补全「规划底线」阈值约定 + 回归基线规模** — 原因：原「规划底线」悬空无值，且「翻译成功率 ≥90%」与质量三指标混淆。明确：词级准确率为相对阻断（降 >2pp）、可复现率 ≥99% 为硬门槛、翻译成功率 90% 为操作性发布门槛，三者并列；并统一基线规模为 CI ≥2000 / 月抽检 ≥200（消除与 Dev-Ready、Sprint 计划的口径差异）。**核心信息未变，仅消除歧义。**
3. **§11 流程图标注方向范围** — 原因：原图将英→火/火→英 与汉↔火 并列为首页入口，易误读为 MVP 范围；现明确英语方向属 V1.1（R08/R09）。
4. **R14 优先级 `P0/P1` → `P0` + 加优先级说明** — 原因：结论卡与 Dev-Ready 均按 P0 交付；将歧义统一为 P0，并注明 R15 自定义规则为 P2。

### 📄 `product-strategy/sprint-plan-v1.0-2026-07-06.md`
5. **修复 SSOT 失效引用** — 原因：原文指向不存在的绝对路径；改为同目录 `dev-ready-mars-translator-2026-07-06.md`，并显式说明「文档§X」映射到该文档的 §1–§7，使依赖挂接全部可解析。
6. **新增「估算合计 82 SP」说明** — 原因：原文件无合计，且历史「对齐记录」提及 200→2000 修正但 PRD 无对应项；现公布 T01–T18 求和 = 82 SP，并标注与 PRD「33 人日」单位不同仅作量级印证。

### 📄 `product-strategy/dev-ready-mars-translator-2026-07-06.md`
7. **`约 88 SP` → `任务点合计 82 SP`** — 原因：原数值与任务表实际求和不符（82），修正为准确值。
8. **资源需求栏补充 SP↔人日换算说明** — 原因：原「一致」说法无依据；改为说明单位不同、按 ~0.4 人日/SP 折算仅作量级印证。
9. **第二部分加「与独立 Sprint 计划文件关系」说明** — 原因：消除两份 Sprint 计划的维护漂移风险，确立 sprint-plan 为权威版、本部分为其镜像。
10. **DoD 与 §7 补全「规划底线」阈值约定** — 原因：与 PRD §5 对齐，明确词级准确率基线由数析在回归基线建立时给定、采用相对阻断。

### 📄 `mars-translator-design-system.md`
11. **§4.1 增加令牌权威性说明** — 原因：明确暗色令牌为唯一权威取值，并要求原型/实现严格采用，为 HTML 暗色修复提供依据。
12. **§10 增加「V1 对照说明」** — 原因：原表「V1」无出处易误解为已有文档；注明其为概念基线，差异来自设计评审。

### 📄 `mars-translator-ui-design.html`
13. **`[data-theme="dark"]` 补全 7 个暗色令牌** — 原因：对齐 design-system §4.1，修复暗色下品牌/语义色混用浅色值的问题（如 `--brand` 暗色应为 `#FF5C95`）。
14. **设置页「主题」分段控件接 JS 并与侧栏联动** — 原因：原设置页主题切换无响应，与侧栏主题按钮不一致；现通过 `data-role="theme"` + `applyTheme` 实现双向同步（含 localStorage 持久化）。

> 说明：HTML 中设置页「强调色」（品红·紫 / 青·蓝）为设计提案，尚未进入 design-system 令牌表，故保留为视觉态、未接功能，避免引入无规范的令牌。
